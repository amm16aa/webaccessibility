Coding for Accessible Design

By: Alex Moerschbacher

Elements

Avoid using non-semantic elements as much as possible when creating your website with HTML. <div> and <span> elements tell screen readers nothing about the content being presented and don’t assist the screen reader in giving context of the webpage to end users. Rather, use semantic elements such as <form>, <table>, and <article> these elements correctly define the context to the screen reader, allowing screen readers to correctly read the information to users. Another reason to use semantic elements is when using keyboard only navigation defining a button as a button (<button>) makes it clickable while using a <div> tag does not. This makes your job as the designer easier as it reduces the amount of work you need to do while making your website accessible.

Headings

The <h1> through <h6> tags are meant to be used to define headings in HTML. Headings are used to break your page into sections and allow search engines and screen readers to understand the structure of your page. Using the headings to make your text BIG or bold will have the same impact on the look of your page but will confuse screen readers and search engines about the layout of your page. Rather use the <strong> tag to make your text bold as it is a semantic tag.

Alternative Text

When placing images onto your website make sure to include an alt attribute to use it when describing an image. If a user is using a screen reader and an alt attribute on the page the screen reader will read the description provide of the image. This allows users with visual impartments to be able to understand what is taking place in the image.

Web Content Accessibility Guidelines

The World Wide Web Consortium (W3C) strives to create a web that is fully accessible for people with a wide range of disabilities. The W3C provides a list of guidelines on their website to create fully accessible web pages. These include the importance of a page title, Image text alternatives, rules for text, interaction and general design choices for creating a accessible website. While this website sets the foundation for a good design choices for accessible websites the W3C guidelines provides one the most complete lists you will be able to find.
